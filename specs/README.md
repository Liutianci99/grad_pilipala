# specs/ — 规范驱动开发

## 流程
1. Boss 写 `requirements.md` — 需求（WHAT + WHY）
2. Pilipala 写 `api.md` — 后端接口设计
3. Pilipala 写 `front_design.md` — 前端页面设计
4. Boss 审核 → 通过 / 修改
5. Pilipala 写代码

## 目录结构
```
specs/
├── constitution.md              # 项目宪法
├── 001-功能名/
│   ├── requirements.md          # Boss 写的需求
│   ├── api.md                   # 后端接口设计
│   └── front_design.md          # 前端页面设计
├── 002-功能名/
│   └── ...
```
