# specs/ — 项目规范

## 文件说明

| 文件 | 内容 |
|------|------|
| `constitution.md` | 项目宪法（核心原则、技术栈、约束） |
| `requirements.md` | 功能需求（按角色划分，每个页面的功能说明） |
| `rule.md` | 编码规范（命名、风格、Git 约定） |
| `api.md` | 后端接口规范（URL、参数、返回格式） |
| `frontend_design.md` | 前端设计规范（颜色、布局、共享组件） |

## 怎么用
- 写新功能前，先看这五个文件
- Boss 提需求 → 更新 requirements.md → Pilipala 按规范写代码
- 规范有变 → 更新对应文件
