# specs/ — 项目规范

## 文件说明

| 文件 | 内容 | 谁写 |
|------|------|------|
| `constitution.md` | 项目宪法（核心原则、技术栈、约束） | 一起定 |
| `rule.md` | 编码规范（命名、风格、约定） | Pilipala |
| `api.md` | 后端接口规范（URL、参数、返回格式） | Pilipala |
| `frontend_design.md` | 前端设计规范（组件、样式、页面结构） | Pilipala |

## 怎么用
- 写新功能前，先看这四个文件
- Boss 提需求 → Pilipala 按规范写代码
- 规范有变 → 更新对应文件
